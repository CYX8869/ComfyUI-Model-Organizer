## 中文

首先，感谢你考虑为这个项目做出贡献！🎉

### 📋 行为准则

参与本项目的所有人都应遵守我们的行为准则：

- 保持友好和专业
- 尊重不同的观点和经验
- 接受建设性的批评
- 关注对社区最有利的事情
- 对其他社区成员表示同理心

### 🚀 如何贡献

#### 1. 报告 Bug

如果你发现了 bug，请通过 GitHub Issues 报告：

1. 搜索现有 Issues，确认是否已有相同问题
2. 使用 Bug Report 模板创建新 Issue
3. 包含以下信息：
   - 详细的问题描述
   - 复现步骤
   - 预期行为
   - 实际行为
   - 系统环境（OS、Python 版本、ComfyUI 版本）
   - 截图（如果适用）

#### 2. 提交功能建议

有好的想法？欢迎提交功能建议：

1. 搜索现有 Issues/Discussions
2. 使用 Feature Request 模板
3. 详细描述：
   - 功能的用例
   - 预期的工作方式
   - 为什么这个功能对大多数用户有用

#### 3. 提交代码 (Pull Request)

**准备工作：**

1. Fork 本仓库
2. 克隆到本地：`git clone https://github.com/your-username/ComfyUI-Model-Organizer.git`
3. 创建特性分支：`git checkout -b feature/your-feature-name`
4. 安装开发依赖

**开发规范：**

- 遵循 PEP 8 代码风格
- 添加必要的注释和文档字符串
- 保持向后兼容性
- 更新相关文档

**提交 PR：**

1. 确保代码通过所有测试
2. 更新 README.md（如需要）
3. 更新 CHANGELOG.md
4. 创建 Pull Request，使用 PR 模板
5. 等待代码审查

### 📝 开发流程

```bash
# 1. Fork 并克隆
git clone https://github.com/your-username/ComfyUI-Model-Organizer.git
cd ComfyUI-Model-Organizer

# 2. 创建分支
git checkout -b feature/amazing-feature

# 3. 提交更改
git commit -m 'Add some amazing feature'

# 4. 推送到分支
git push origin feature/amazing-feature

# 5. 打开 Pull Request
```

### 🎯 适合新手的任务

如果你是第一次贡献，可以查看带有 `good first issue` 标签的 Issues。

## English

First off, thanks for taking the time to contribute! 🎉

### 📋 Code of Conduct

Everyone participating in this project is expected to abide by our Code of Conduct:

- Be friendly and professional
- Respect differing viewpoints and experiences
- Gracefully accept constructive criticism
- Focus on what is best for the community
- Show empathy towards other community members

### 🚀 How to Contribute

#### 1. Reporting Bugs

If you find a bug, please report it via GitHub Issues:

1. Search existing Issues to confirm the problem hasn't been reported
2. Create a new Issue using the Bug Report template
3. Include:
   - Detailed problem description
   - Steps to reproduce
   - Expected behavior
   - Actual behavior
   - System environment (OS, Python version, ComfyUI version)
   - Screenshots (if applicable)

#### 2. Feature Requests

Have a great idea? Feel free to submit feature suggestions:

1. Search existing Issues/Discussions
2. Use the Feature Request template
3. Describe in detail:
   - Use cases for the feature
   - How it would work
   - Why this feature would be useful to most users

#### 3. Submitting Code (Pull Request)

**Preparation:**

1. Fork this repository
2. Clone locally: `git clone https://github.com/your-username/ComfyUI-Model-Organizer.git`
3. Create feature branch: `git checkout -b feature/your-feature-name`
4. Install development dependencies

**Development Guidelines:**

- Follow PEP 8 code style
- Add necessary comments and docstrings
- Maintain backward compatibility
- Update relevant documentation

**Submitting PR:**

1. Ensure code passes all tests
2. Update README.md (if needed)
3. Update CHANGELOG.md
4. Create Pull Request using the PR template
5. Wait for code review

### 📝 Development Workflow

```bash
# 1. Fork and clone
git clone https://github.com/your-username/ComfyUI-Model-Organizer.git
cd ComfyUI-Model-Organizer

# 2. Create branch
git checkout -b feature/amazing-feature

# 3. Commit changes
git commit -m 'Add some amazing feature'

# 4. Push to branch
git push origin feature/amazing-feature

# 5. Open Pull Request
```

### 🎯 Good for Beginners

If this is your first contribution, check out Issues labeled `good first issue`.
